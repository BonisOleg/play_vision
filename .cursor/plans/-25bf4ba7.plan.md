## Як працюють iOS блоки (приклад з home.css)

**Стандартний медіа-запит:**

```css
@media (max-width: 768px) {
    .hero-section .section-content.hero-new-design {
        left: 5px;
        padding: 0.75rem 1rem 1.5rem 0;
        /* базові стилі */
    }
}
```

**iOS блок (ДОПОВНЕННЯ):**

```css
@supports (-webkit-touch-callout: none) {
    @media (max-width: 768px) {
        .hero-btn-new {
            /* ТІЛЬКИ webkit властивості */
            -webkit-appearance: none;
            -webkit-tap-highlight-color: transparent;
        }
    }
}
```

**Результат:** iOS блок НЕ перезаписує базові стилі, він додає тільки webkit-оптимізації для інших елементів або ті самі елементи але з додатковими webkit властивостями.

## Приклад додавання (для events.css)

**Поточний стан:** Немає iOS блоку

**Що додати в кінець файлу:**

```css
/* ===== iOS SAFARI OPTIMIZATION ===== */
@supports (-webkit-touch-callout: none) {
    /* Кнопки подій */
    .event-btn,
    .calendar-btn,
    .btn-details {
        -webkit-tap-highlight-color: transparent;
        -webkit-touch-callout: none;
    }
    
    /* Календар на малих екранах */
    @media (max-width: 768px) {
        .calendar-container {
            -webkit-overflow-scrolling: touch;
        }
    }
}
```

**Вплив:** Тільки на iOS - прибирає синє виділення при натисканні, покращує скролінг. На Android/Desktop - БЕЗ ЗМІН.
